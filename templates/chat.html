System Architecture
Full RAG pipeline — from PDF upload to Qwen3 answer
Ingestion Pipeline
1. PDF Upload
Flask receives file → saved to uploads/
2. Text Extraction — PyMuPDF
Fast native text extraction per page
3. OCR Fallback — PaddleOCR
Scanned/image pages → rendered at 2× and OCR'd
4. Table Extraction — Docling
Structured tables → Markdown representation
5. Image Extraction — PyMuPDF + PaddleOCR
Embedded images extracted → OCR'd for text
6. Chunking
512-char chunks with 80-char overlap + metadata
7. Embedding — BAAI/bge-m3
1024-dim dense vectors, normalised for cosine
8. FAISS Index — IndexFlatIP
Inner-product search, persisted to disk
RAG Query Pipeline
1. User Query
Natural language question from UI
2. Query Embedding — BAAI/bge-m3
Same model used for ingestion
3. FAISS Retrieval — Top 20
Fast approximate nearest-neighbour search
4. Re-ranking — bge-reranker-large
Cross-encoder re-scores; top 5 retained
5. Prompt Assembly
System prompt + context chunks + user query
6. LLM — Qwen3 via Ollama
Grounded answer with page citations
7. Log → chat_history.txt
Q, A, and source citations persisted
